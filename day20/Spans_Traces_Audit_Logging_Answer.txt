Homework: How will you relate spans/traces with audit logging, when to
use what?

Answer

Spans/Traces and Audit Logs serve different purposes but complement each
other in a production AI system.

1.  Spans/Traces

-   Used for monitoring and debugging application execution.
-   Capture execution flow, latency, errors, and dependencies.
-   Temporary observability data.
-   Useful for developers and SRE teams.

2.  Audit Logs

-   Used for recording permanent business and security events.
-   Capture user actions, model decisions, timestamps, and compliance
    information.
-   Tamper-evident and retained for investigations.
-   Useful for auditors, security teams, and compliance.

Relationship

Every audit log entry can include a Trace ID and Span ID generated by
OpenTelemetry.

Execution Flow

User Request | Generate Trace ID | OpenTelemetry Spans | Research Agent
| Summary Agent | Judge Agent | Create Audit Log (Store Trace ID + Span
ID + Hash) | Save to JSONL / Database

When to Use Spans/Traces - Measure execution time. - Debug failures. -
Monitor distributed systems. - Visualize request flow.

When to Use Audit Logs - Record user actions. - Store AI decisions. -
Maintain compliance. - Support forensic investigations.

Conclusion

Spans answer: “How did the system execute?” Audit logs answer: “What
happened, who did it, and what decision was made?”

By storing Trace ID and Span ID in audit logs, organizations can
correlate runtime execution with permanent business records, providing
complete observability, debugging, security, and compliance.
