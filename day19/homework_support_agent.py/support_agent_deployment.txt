1. COMPREHENSIVE ARCHITECTURE NOTE & COMPONENT RATIONALE
--------------------------------------------------------------------------------

The support agent is engineered around an Event-Driven State-Machine topology. 
Unlike basic linear chains, human customer support conversations are non-linear, 
dynamic, require iterative tool chaining, multi-layered memory recall, and strict 
safety checkpoints before executing state-mutating high-risk actions.

Core Layers & Why We Use Them:

A. Short-Term Memory State (Redis Chat Message History)
   - WHY: LLMs are natively stateless. To provide conversational continuity, the 
     agent must remember recent history across distinct API calls, horizontal scaling 
     layers, or server restarts without relying on memory-bound global variables.
   - HOW: We use Redis via RedisChatMessageHistory mapped to a unique 'thread_id'.
     It delivers ultra-low latency memory access and enterprise-grade state persistence.

B. Long-Term Memory & Recall Store (ChromaDB Vector Store)
   - WHY: Historical information (e.g., user preferences written months ago, old ticket 
     resolutions) cannot fit inside short-term conversational windows without causing 
     massive context window inflation, cost inefficiencies, and token degradation.
   - HOW: Important facts are vectorized using OpenAI Embeddings and pushed into 
     ChromaDB. The agent uses 'query_long_term_memory' to fetch semantic context only when needed.

C. Multi-Tool Orchestration & Chaining Layer (The 23-Tool Belt)
   - WHY: Support agents must execute complex tasks against diverse internal APIs 
     (Billing, Shipping, Inventory, Auth). Modularity requires isolating these actions.
   - HOW: We construct and register 23 distinct tool definitions. LangGraph handles 
     the execution loop: the agent calls a tool, gets the result, returns to the 
     central 'agent' node to evaluate the output, and decides whether to chain another 
     tool or reply.

D. Async Queue-Backed Processing (Celery + Redis Broker)
   - WHY: High-overhead actions like running security database compliance audits take 
     significant processing time. Blocking the primary LLM conversation thread while 
     waiting for an expensive operation kills the user experience and risks HTTP timeouts.
   - HOW: The 'trigger_async_account_audit' tool pushes a background task onto Celery via 
     a Redis broker and returns a tracking 'Job ID' immediately, maintaining system fluidity.

E. Human-In-The-Loop (HITL) Validation Gate
   - WHY: High-risk operations (such as processing cash refunds or editing access tokens) 
     cannot be delegated fully to automated LLM logic without introducing large risk surfaces.
   - HOW: Using LangGraph's 'interrupt_before' parameter, the graph freezes state execution 
     natively before entering the 'human_approval_gate' node. The process pauses, 
     saves its checkpoint, and waits for a supervisor override payload (APPROVED/DENIED).

F. Prompt Caching, Resiliency, & Retries
   - WHY: Upstream LLM network jitter or tool API timeouts shouldn't break an active 
     user's customer support interaction.
   - HOW: We apply the 'tenacity' framework to wrap core LLM invocations with exponential 
     backoffs. Prompt caching optimization is maintained by placing static system prompts 
     strictly at the beginning of the chat layout matrix.

import uuid
from langchain_core.tools import tool
from database import vector_store
from tasks import run_deep_account_audit

# --- Core Context Memory Tools ---
@tool
def query_long_term_memory(query: str) -> str:
    """Search historical customer resolutions, permanent operational updates, or persistent notes."""
    docs = vector_store.similarity_search(query, k=2)
    if not docs:
        return "No permanent historical memories found matching this query."
    return "\n---\n".join([d.page_content for d in docs])

@tool
def save_to_long_term_memory(memory_text: str) -> str:
    """Permanently store critical, structural facts about a customer for future interactions."""
    vector_store.add_texts([memory_text])
    return "Successfully committed to long term vector memory."

# --- Async Queue-Backed Tool ---
@tool
def trigger_async_account_audit(user_id: str) -> str:
    """Dispatches an intensive compliance profile audit to the background queue. Returns job tracking ID."""
    # Enqueue task non-blockingly to Celery worker cluster
    task = run_deep_account_audit.delay(user_id)
    return f"Task safely enqueued to Redis broker. Tracking Job ID: {task.id}. Let the user know we're evaluating."

# --- Human-Gate Controlled Tool ---
@tool
def issue_refund(user_id: str, amount_usd: float) -> str:

from typing import Annotated, Literal
from typing_extensions import TypedDict
from tenacity import retry, stop_after_attempt, wait_exponential

from langchain_core.messages import SystemMessage, AIMessage, ToolMessage
from langgraph.graph.message import add_messages
from langchain_openai import ChatOpenAI

from langgraph.graph import StateGraph, START, END
from langgraph.checkpoint.memory import MemorySaver

from tools import TOOL_BELT, TOOL_MAP

# 1. State Definition
class AgentState(TypedDict):
    messages: Annotated[list, add_messages]
    action_approved: bool  # Intercept flag managed by Human Gate actions

# 2. Resilient LLM Interface
@retry(wait=wait_exponential(min=1, max=10), stop=stop_after_attempt(3))
def call_llm_with_retry(messages):
    """Invokes Model. Static system message anchor optimizes for Prompt Caching."""
    llm = ChatOpenAI(model="gpt-4o", temperature=0)
    return llm.bind_tools(TOOL_BELT)(messages)

# 3. Graph Logic Routines
def agent_router(state: AgentState) -> Literal["human_approval_gate", "execute_tools", "__end__"]:
    last_message = state["messages"][-1]
    if not last_message.tool_calls:
        return END
    
    for tool_call in last_message.tool_calls:
        if tool_call["name"] == "issue_refund":
            return "human_approval_gate"
            
    return "execute_tools"

def call_agent_node(state: AgentState):
    system_anchor = SystemMessage(
        content="You are an expert internal AI assistant. You have access to real-time tools, "
                "long term historical memory bases, and non-blocking background queue pipelines."
    )
    response = call_llm_with_retry([system_anchor] + state["messages"])
    return {"messages": [response], "action_approved": False}

def human_approval_gate_node(state: AgentState):
    """Execution breakpoint node. The graph freezes cleanly BEFORE executing this node block."""
    pass

def execute_tools_node(state: AgentState):
    messages = state["messages"]
    last_message = messages[-1]
    tool_outputs = []
    
    for tool_call in last_message.tool_calls:
        name = tool_call["name"]
        args = tool_call["args"]
        
        # Check human supervisor validation metrics
        if name == "issue_refund" and not state.get("action_approved", False):
            output_msg = ToolMessage(
                content="Rejected: Operational execution aborted. Action lacks verified human manager approval signature.",
                tool_call_id=tool_call["id"]
            )
        else:
            executed_result = TOOL_MAP[name].invoke(args)
            output_msg = ToolMessage(content=str(executed_result), tool_call_id=tool_call["id"])
            
        tool_outputs.append(output_msg)
        
    return {"messages": tool_outputs}

# 4. Assembling Graph Machine Architecture Topology
workflow = StateGraph(AgentState)
workflow.add_node("agent", call_agent_node)
workflow.add_node("human_approval_gate", human_approval_gate_node)
workflow.add_node("execute_tools", execute_tools_node)

workflow.set_entry_point("agent")

workflow.add_conditional_edges(
    "agent",
    agent_router,
    {
        "human_approval_gate": "human_approval_gate",
        "execute_tools": "execute_tools",
        END: END
    }
)
workflow.add_edge("execute_tools", "agent")
workflow.add_edge("human_approval_gate", "execute_tools")

# Compile with cross-session persistence checker to handle runtime interruptions
agent_application = workflow.compile(
    checkpointer=MemorySaver(),
    interrupt_before=["human_approval_gate"]
)

from agent import agent_application
from database import get_short_term_history
from langgraph.errors import GraphInterrupt

def run_simulation():
    # Setup state execution tracing definitions
    thread_config = {"configurable": {"thread_id": "session_cus_5510"}}
    history = get_short_term_history("session_cus_5510")
    history.clear()

    # --- Scenario 1: Complex tool loop chaining + background task dispatching ---
    user_input = "Hello! Please save that I am an Enterprise Tier client to my long-term memory, verify my order status, and initialize a system database background audit profile."
    print(f"\n[USER PROMPT]: {user_input}\n")
    history.add_user_message(user_input)

    for event in agent_application.stream({"messages": history.messages}, thread_config):
        for node, data in event.items():
            print(f"-> Active State Transition Node: {node}")
            if "messages" in data:
                print(f"Result Output Data: {data['messages'][-1].content}\n")

    # --- Scenario 2: High-Risk Action Invoking Human Approval Interrupt ---
    critical_input = "I require an immediate refund of $350.00 to my Visa card for this defective item."
    print(f"\n[USER PROMPT]: {critical_input}\n")
    history.add_user_message(critical_input)

    try:
        # The runtime execution state will trigger an exception, saving the thread's place
        for event in agent_application.stream({"messages": history.messages}, thread_config):
            print(event)
    except GraphInterrupt:
        print("\n=== [SYSTEM HITL INTERRUPT] State machine frozen: High Risk Operation Detected ===")
        
        # Simulate supervisor console action override decision
        supervisor_decision = "APPROVED"
        print(f"[Supervisor Dashboard Override Command Issued]: {supervisor_decision}")
        
        if supervisor_decision == "APPROVED":
            # Apply security clearance state properties directly onto the thread checkpoint
            agent_application.update_state(thread_config, {"action_approved": True}, as_node="human_approval_gate")
            print("=== [Graph Context Resuming] Credentials Authorized. Enacting mutation step ===\n")
            
            # Restart graph loop execution seamlessly
            for event in agent_application.stream(None, thread_config):
                for node, data in event.items():
                    print(f"-> Active State Transition Node: {node}")
                    if "messages" in data:
                        print(f"Result Output Data: {data['messages'][-1].content}\n")

if __name__ == "__main__":
    run_simulation()
